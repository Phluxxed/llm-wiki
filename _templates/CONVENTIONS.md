## Sources Layer

`sources/` holds raw, unmodified inputs exactly as they came in. These are never edited.

Every source file must start with this header block:

```
> **Source type:** Jira ticket | Confluence page | Slack thread | Meeting notes | Authored
> **URL:** (if applicable)
> **Fetched:** YYYY-MM-DD
> ‚öÝÔ∏è Do not edit this file. Edit the derived wiki page instead.
```

Not every wiki page needs a source file ‚Äî authored pages are fine without one. But anything derived from an external artifact must have one.

When adding a new source:

1. Save it to `sources/` with a descriptive filename
2. Derive the wiki page using `_templates/{PAGE_TYPE_SLUG}.md`
3. Set `source: sources/filename.md` in the wiki page frontmatter

## Template

Copy `_templates/{PAGE_TYPE_SLUG}.md` as the starting point for every new {PAGE_TYPE_SINGULAR}.

## Keeping the Wiki Healthy

When adding or significantly updating a page:

1. Update `index.md` ‚Äî add or revise the one-liner for that page
2. Append a row to `log.md` ‚Äî `## [YYYY-MM-DD] action | detail`
3. Add a `Related` link in the new page if it connects to an existing one

Mandatory sections (always include): **What This Is**, **How It Works**, **Risk Register**, **Prerequisites**

Optional sections (include only if relevant): Architecture, Authentication, MCP Servers, CI/CD Setup, Output, Setup, Usage, Notes & Lessons Learned
